Actividad 02
Crear Base de Datos GradosTitulos
Ejecutar el script SQL de creación de la base de datos en Azure SQL Database o SQL Server local.
Dos opciones: Puedes crear la BD en Azure SQL Database (usando tus créditos de estudiante) o en tu instancia local de SQL Server 2022 conectada a Azure. Se mostrará el proceso completo para ambas opciones.

Script de creación — CreaBaseDatosGradosTitulos.sql

T-SQL · SQL Server 2022 
-- ============================================================
-- BASE DE DATOS: UPLA - GRADOS Y TÍTULOS DE PREGRADO
-- Normalización 3FN - Reglamento General de Grados y Títulos
-- Motor: SQL Server 2022
-- Autor: Ing. Raúl Fernández Bejarano
-- Fecha: 02/06/2026
-- ============================================================

USE master;
GO

-- Eliminar si existe
IF EXISTS (SELECT name FROM sys.databases WHERE name = N'GradosTitulos')
BEGIN
    ALTER DATABASE GradosTitulos SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
    DROP DATABASE GradosTitulos;
END
GO

-- Crear base de datos con archivos configurados
CREATE DATABASE GradosTitulos
ON PRIMARY (
    NAME     = GradosTitulos_Data,
    FILENAME = 'D:\BaseDatos2026\GradosTitulos_data.mdf',
    SIZE     = 20MB,
    MAXSIZE  = 20GB,
    FILEGROWTH = 5MB
)
LOG ON (
    NAME     = GradosTitulos_Log,
    FILENAME = 'D:\BaseDatos2026\GradosTitulos_log.ldf',
    SIZE     = 6MB,
    MAXSIZE  = 2GB,
    FILEGROWTH = 2MB
);
GO
1

Crear la carpeta destino en tu equipo

Antes de ejecutar el script, crea la carpeta donde se guardarán los archivos de la BD. Abre PowerShell como Administrador y ejecuta:

PowerShell
# Crear directorio para la base de datos
New-Item -ItemType Directory -Force -Path "D:\BaseDatos2026"

# Verificar que existe
Test-Path "D:\BaseDatos2026"
Si usas Azure SQL Database en la nube, omite este paso — Azure gestiona el almacenamiento automáticamente.
2

Abrir SQL Server Management Studio (SSMS)

Conecta a tu instancia SQL Server y abre una nueva consulta (Ctrl + N). Asegúrate de estar conectado como Administrador del servidor.

3

Ejecutar el script y verificar creación

Pega el script completo en SSMS y presiona F5 para ejecutarlo. Al completarse, verifica la BD con:

T-SQL
-- Verificar que la BD fue creada
SELECT
    name            AS BaseDatos,
    create_date     AS FechaCreacion,
    state_desc      AS Estado,
    recovery_model_desc AS ModeloRecuperacion,
    compatibility_level AS NivelCompatibilidad
FROM sys.databases
WHERE name = N'GradosTitulos';
GO

-- Ver propiedades del archivo de datos
USE GradosTitulos;
GO
SELECT name, physical_name, size/128 AS TamanioMB, state_desc
FROM sys.database_files;
Si el resultado muestra GradosTitulos con estado ONLINE, la base de datos fue creada exitosamente.

Paso 3 — Vistas del Sistema

T-SQL · 3 Vistas
-- V01: Condición de egresado (elimina dependencia transitiva)
CREATE OR ALTER VIEW Academico.V_CondicionEgresado
AS
    SELECT
        m.IdMatricula,
        p.DNI,
        p.Nombres + ' ' + p.ApellidoPaterno AS NombreCompleto,
        pe.NombrePrograma,
        m.CicloEgreso,
        m.FechaEgreso,
        m.CondicionEgresado
    FROM Academico.Matricula m
    JOIN Academico.Persona p ON p.IdPersona = m.IdPersona
    JOIN Academico.ProgramaEstudios pe ON pe.IdPrograma = m.IdPrograma;
GO

-- V02: Estado actual del trabajo de investigación
CREATE OR ALTER VIEW Tramite.V_EstadoTrabajo
AS
    SELECT
        ti.IdTrabajo,
        ti.Titulo,
        ti.TipoTrabajo,
        et.Descripcion AS EstadoTramite,
        t.FechaApertura
    FROM Tramite.TrabajoInvestigacion ti
    JOIN Tramite.Tramite t ON t.IdTramite = ti.IdTramite
    JOIN Catalogo.EstadoTramite et ON et.IdEstadoTramite = t.IdEstadoTramite;
GO

-- V03: Nota final de sustentación calculada
CREATE OR ALTER VIEW Evaluacion.V_NotaFinalSustentacion
AS
    SELECT
        s.IdSustentacion,
        s.IdTrabajo,
        s.NroIntento,
        s.NotaExposicion,
        s.NotaDefensa,
        ROUND((s.NotaExposicion + s.NotaDefensa) / 2.0, 1) AS NotaFinalCalculada,
        s.Condicion AS CondicionRegistrada,
        s.NroActa
    FROM Evaluacion.Sustentacion s;
GO

Paso 4 — Funciones Auxiliares

T-SQL · 2 Funciones
-- F01: Ciclo mínimo para presentar plan de tesis (Art. 22°)
CREATE OR ALTER FUNCTION Academico.F_CicloMinimoPlan
(
    @NumSemestres TINYINT
)
RETURNS TINYINT
AS
BEGIN
    RETURN CASE @NumSemestres
        WHEN 10 THEN 9   -- Carrera 10 semestres → plan desde semestre 9
        WHEN 12 THEN 11  -- Carrera 12 semestres → plan desde semestre 11
        WHEN 14 THEN 13  -- Carrera 14 semestres → plan desde semestre 13
        ELSE NULL
    END;
END;
GO

-- F02: Verificar si similitud es aceptable (Art. 7°: máximo 30%)
CREATE OR ALTER FUNCTION Evaluacion.F_SimilitudAceptable
(
    @Porcentaje DECIMAL(5,2)
)
RETURNS BIT
AS
BEGIN
    RETURN CASE WHEN @Porcentaje < 30.00 THEN 1 ELSE 0 END;
END;
GO

-- Ejemplos de uso:
SELECT Academico.F_CicloMinimoPlan(10)  -- Retorna: 9
SELECT Evaluacion.F_SimilitudAceptable(28.50)  -- Retorna: 1 (aceptable)
SELECT Evaluacion.F_SimilitudAceptable(31.00)  -- Retorna: 0 (supera límite)

Paso 5 — Stored Procedures

T-SQL · SP01 — Registrar Trámite
CREATE OR ALTER PROCEDURE Tramite.SP_RegistrarTramite
    @IdMatricula    INT,
    @IdTipoTramite  TINYINT,
    @IdCoordinacion TINYINT,
    @IdTramite      INT OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        BEGIN TRANSACTION;

        -- Validar que la matrícula exista
        IF NOT EXISTS (SELECT 1 FROM Academico.Matricula WHERE IdMatricula = @IdMatricula)
            THROW 50001, 'La matrícula especificada no existe.', 1;

        -- Validar que no tenga trámite activo del mismo tipo
        IF EXISTS (
            SELECT 1 FROM Tramite.Tramite
            WHERE IdMatricula = @IdMatricula
              AND IdTipoTramite = @IdTipoTramite
              AND IdEstadoTramite NOT IN (5, 6, 7)
        )
            THROW 50002, 'Ya existe un trámite activo del mismo tipo.', 1;

        INSERT INTO Tramite.Tramite (IdMatricula, IdTipoTramite, IdEstadoTramite, IdCoordinacion)
        VALUES (@IdMatricula, @IdTipoTramite, 1, @IdCoordinacion);

        SET @IdTramite = SCOPE_IDENTITY();
        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        IF @@TRANCOUNT > 0 ROLLBACK TRANSACTION;
        THROW;
    END CATCH;
END;
GO

Preguntas frecuentes

¿Qué pasa si mi correo UPLA no es reconocido por Azure?

Verifica que estés usando el correo institucional oficial completo (@upla.edu.pe). Si sigue sin funcionar, contacta al área de TI de la UPLA para confirmar que tu institución está registrada en el programa Azure for Students. También puedes intentar con una VPN o en un navegador diferente.

¿Cuánto cuesta crear una Azure SQL Database con créditos de estudiante?

El tier más económico (Basic, 5 DTUs, 2 GB) cuesta aproximadamente $0.0067/hora ≈ $5 USD/mes. Con tus $100 USD de crédito tienes aproximadamente 20 meses de uso continuo. Para un proyecto académico que no necesita estar 24/7, puedes pausar la BD cuando no la uses para maximizar el crédito.

¿El path 'D:\BaseDatos2026' funciona en Azure?

No. En Azure SQL Database no puedes especificar rutas de archivos locales — Azure gestiona el almacenamiento automáticamente. El script CREATE DATABASE con rutas locales se usa solo para SQL Server instalado en tu máquina o VM. Para Azure SQL Database, simplemente usa el portal de Azure o los comandos T-SQL sin las cláusulas ON PRIMARY y LOG ON.

¿Puedo conectar SSMS local a mi Azure SQL Database?

Sí. En SSMS, al conectar escribe el nombre del servidor en formato: tu-servidor.database.windows.net. Selecciona autenticación SQL Server y usa las credenciales que configuraste al crear el servidor en Azure. Asegúrate de agregar tu IP a las reglas de firewall del servidor en el portal de Azure.
